# What is a schema?
## 3 inputs to Schema Design 1. The data your application needs. 2. Your application's read usage of the data. 3. Your application's write usage of the data.
## Why is schema design important? Schema is defined at the application-level Flexible schema benefits Development phase is more agile with a flexible schema Good schema needed for production Schema evolutions possible to meet future business needs
## MongoDB is a document database Documents are stored in BSON Each document has keys and values
## Library web application Different schemas are possible
## Author schema 
{
    "_id": int,
    "first_name": string,
    "last_name": string
}
## User schema 
{
    "_id": int,
    "username": string,
    "password": string
}
## Book schema 
{
    "_id" : int,
    "author" : int,
    "available" : boolean,
    "created_at" : date,
    "created_by" : date,
    "isbn" : string,
    "language" : string,
    "pages" : int,
    "publisher" : {
        "city" : string,
        "name" : string
    },
    "slug" : string,
    "subjects" : [
         string,
         string
   ],
    "title" : string
}
